# SeoulMySoulMate 상세 기능 요구사항

## 1. 장소 중심 커뮤니티

- 새 게시글은 관광 콘텐츠 한 곳을 선택해 연결한다. 기존의 장소 미연결 글은 계속 조회할 수 있다.
- 커뮤니티 목록은 연결 장소의 대표 이미지, 장소명, 카테고리, 주소와 글 요약을 함께 표시한다.
- 사용자는 게시글 제목/내용뿐 아니라 장소명/주소, 관광 카테고리로 게시글을 필터링할 수 있다.
- 후기가 많은 장소를 별도 영역에 노출하고, 선택하면 해당 장소의 글만 조회한다.
- 커뮤니티 글 작성 시 장소를 검색해 연결할 수 있고, 콘텐츠 상세에서 진입하면 자동 선택된다.
- 장소가 없는 과거 자유 글은 "서울 자유 이야기"로 구분한다.

### 완료 조건

- `GET /api/posts` 응답에 연결된 `content` 정보와 본문이 포함된다.
- `content_id`, `category`, `keyword` 조건으로 페이지네이션 조회가 가능하다.
- `GET /api/community/places`에서 게시글이 연결된 인기 장소와 글 수를 제공한다.

## 2. SQLite 근거형 Soul Chat

- 사용자의 지역, 관심 분야, 여행시간 표현을 분석한다.
- 답변과 추천 카드는 SQLite에서 검색한 관광 콘텐츠만 근거로 생성한다.
- OpenAI에는 검색된 상위 결과만 전달하고 대화 히스토리를 유지한다.
- 추천 카드에서 상세 보기와 일정 추가가 가능하다.
- `backend/.env`에 `OPENAI_API_KEY`를 추가하면 서버 재시작 없이 다음 요청부터 AI 모드가 활성화된다.
- 키가 없을 때도 SQLite 검색 기반 기본 답변을 제공한다.
- `GET /api/chat/status`로 현재 연결 모드를 확인할 수 있다.

## 3. 라이트/다크 모드

- 헤더에서 테마를 전환하고 선택값을 브라우저에 저장한다.
- 첫 방문에는 운영체제 테마 설정을 따른다.
- 배경, 카드, 입력창, 경계선과 텍스트가 두 테마에서 모두 읽기 쉬워야 한다.

## 4. 전체 장소 지도

- 플래너 지도와 별도로 전체 관광 콘텐츠를 탐색하는 `/map` 화면을 제공한다.
- 좌표가 있는 모든 장소를 표시하고 대량 마커는 클러스터링한다.
- 카테고리, 주소, 장소명으로 필터링할 수 있다.
- 지도 또는 결과 목록에서 콘텐츠 상세로 이동할 수 있다.
- `GET /api/contents/map`은 좌표가 있는 전체 장소를 필터 조건과 함께 제공한다.
